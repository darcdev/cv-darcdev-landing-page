MCP is plumbing. It's not sexy until you have it working.

## 1. Internal specs server

Exposes the repo's specs and ADRs. The agent enters with context, not blind.

## 2. Tickets server

Reads Jira/Linear. The agent can review a ticket before proposing a PR.

## 3. Runbooks server

Incident runbooks accessible to the support agent. The first line answers in seconds.
